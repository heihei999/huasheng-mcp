# 使用场景

当用户提供行测题目，尤其是资料分析、数量关系、判断推理、言语理解与表达题目时，使用此 Skill。

# 必须调用 MCP 工具

解题前必须调用 MCP 工具完成：

1. `classify_question`
2. `search_methods` 或 `get_method_card`
3. 对应模块的 solve 工具
4. 必要时调用 `get_source_reference`

不得直接凭模型记忆答题。

# 模块规则

## 资料分析

- 禁止暴力长除。
- 必须先识别题型：增长率、增长量、基期量、现期量、比重、平均数、倍数等。
- 必须判断选项差距，再决定估算、截位直除、415 份数法、差分法等。
- 输出应说明使用的速算策略。

## 图形推理

- 必须按观察顺序验证。
- 不得凭感觉猜规律。
- 必须先判断图形组成相同、相似还是不同。
- 再依次验证位置、样式、属性、数量、空间重构。
- 最后必须做选项验证。

## 言语理解

- 必须分析文段结构。
- 主旨、意图、标题题不得只凭语感。
- 语句填入与排序必须检查话题一致、代词指代、关联词衔接、时间顺序、逻辑顺序。
- 逻辑填空必须依据语境提示、固定搭配、感情色彩、程度轻重、词义侧重。

## 判断推理

- 逻辑判断必须找论点、论据、论证关系。
- 加强/削弱题必须比较选项力度。
- 定义判断必须拆主体、客体、条件、方式、目的、结果。
- 类比推理必须先判一级关系，再做二级辨析。

# 输出格式

```text
题型识别：
调用方法：method_id + method_name
结构分析 / 题干要素：
解题步骤：
选项排除：
答案：
来源：source_file + source_page
```

若调用 `need_review=true` 卡片，必须提示：

```text
注意：本方法卡片标记 need_review=true，建议人工复核来源页。
```
