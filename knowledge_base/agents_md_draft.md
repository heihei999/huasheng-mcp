# AGENTS.md

## 项目目标

本项目基于 `xingce_huasheng_final_kb/` 实现一个行测方法论解题工具 `xingce-solver`，支持 CLI、MCP Server、Claude Code Skill、Codex 和 opencode。

## 使用知识库规则

1. 先读取 `schema_spec.md`。
2. 再读取 `all_cards.jsonl` 和 `global_router_rules.yaml`。
3. 所有解题必须先分类，再调用方法卡片。
4. 不得把全库一次性塞入提示词。
5. 检索时优先使用 `synonyms.yaml` 扩展查询。
6. 遇到 `need_review=true` 的卡片必须降权，并在输出中提示。

## 必须调用工具

核心工具至少包括：

- `classify_question`
- `search_methods`
- `get_method_card`
- `solve_data_analysis`
- `solve_graphic_reasoning`
- `solve_logic_reasoning`
- `solve_verbal`
- `solve_quantitative`
- `get_source_reference`

## 禁止事项

1. 禁止重新解析 PDF。
2. 禁止重新生成知识卡片。
3. 禁止修改 `all_cards.jsonl`。
4. 禁止暴力精算替代方法论。
5. 禁止资料分析题直接长除。
6. 禁止图形推理凭感觉猜规律。
7. 禁止言语题不分析结构直接选。
8. 禁止逻辑判断不拆论点、论据、论证关系。

## 各模块解题基本流程

### 资料分析

识别统计量 → 读取公式/速算方法 → 抽取现期、基期、增长率、比重等输入 → 判断选项差距 → 估算或计算 → 输出。

### 数量关系

识别题型 → 判断是否可代入排除、赋值、方程、比例、枚举 → 调用对应卡片 → 输出模型与步骤。

### 判断推理-图形推理

调用总流程卡 → 判断组成相同/相似/不同 → 位置/样式/属性/数量/空间重构 → 选项验证。

### 判断推理-逻辑判断

识别加强/削弱/前提假设/解释/评价/推出 → 拆论点、论据、论证方式 → 比较选项力度 → 输出。

### 判断推理-定义判断

抓被定义词 → 拆主体、客体、条件、方式、目的、结果 → 按属于/不属于方向排除。

### 判断推理-类比推理

先判语义/语法/逻辑/对应一级关系 → 再做种属/组成、矛盾/反对、特指/泛指等二级辨析。

### 言语理解

主旨意图：分析文段结构 → 找主体词、主题句、对策或观点 → 排除干扰项。

语句表达：检查话题一致、代词指代、关联词衔接、首尾句、逻辑顺序。

逻辑填空：找语境提示、搭配、感情色彩、程度轻重、词义侧重。

## 输出格式

```text
题型识别：
调用方法：
关键信息：
解题步骤：
选项分析：
答案：
来源：
```

## 代码开发注意事项

- 使用流式 JSONL 读取，避免一次性构造超大 prompt。
- 建立 method_id 索引。
- 路由规则中所有 method_id 必须能在 all_cards 中找到。
- BM25/关键词检索只负责召回，最终必须由方法卡片约束解题。
- 单元测试至少覆盖：JSONL 解析、YAML 解析、id 唯一性、路由引用、need_review 降权。
