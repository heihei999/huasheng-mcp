# 言语理解-主旨意图方法库

## 题型总览

本模块面向片段阅读中的主旨意图类题目，核心原则是：**先做文段结构分析，再做选项验证**，不能只凭语感或关键词硬选。

| 题型 | 常见问法 | 优先调用卡片 |
|---|---|---|
| 主旨概括 | 主要说明、主要介绍、主要讲述、概括最准确 | `vr_mi_total_flow_001`, `vr_mi_main_summary_001`, `vr_mi_center_understanding_001` |
| 意图判断 | 意在说明、旨在强调、重在说明、想表达 | `vr_mi_total_flow_001`, `vr_mi_intention_solution_001`, `vr_mi_problem_solution_001` |
| 标题填入 | 最适合做标题、作为标题最恰当 | `vr_mi_title_total_001`, `vr_mi_title_keyword_001` |
| 中心理解 | 中心意思、核心观点、这段文字主要讲了 | `vr_mi_total_flow_001`, `vr_mi_center_understanding_001` |

## 总体解题流程

1. **识别题干问法**：先判断是主旨概括、意图判断、标题填入还是中心理解。
2. **划分文段结构**：至少标出“引出话题/问题、分析论证、观点/对策、结论”。
3. **锁定主体词**：首句主语、高频词、重点句主语必须和正确选项一致。
4. **判定重点层**：
   - 有对策：优先选对策。
   - 有作者观点：优先选观点。
   - 无观点对策但有问题：概括问题或原因。
   - 并列结构：整体概括。
   - 标题题：先确定主旨，再压缩成准确、简洁、醒目的标题。
5. **选项排除**：重点排除出处不对、偷换概念、范围扩大/缩小、绝对化、无中生有、杂糅、太中性等。

对应总流程卡片：`vr_mi_total_flow_001`。

## 公式体系 / 结构模式

### 1. 问题—对策结构

- **结构**：提出问题 → 分析问题 → 给出对策
- **适用场景**：文段出现风险、困境、弊端、现实问题，后文出现“应、需要、必须、为此、只有……才”等。
- **重点**：对策。
- **不适用情况**：选项中的对策无法解决文段问题，或主语与问题主体不一致。
- **常见错误**：原文已有对策，却选择只描述问题的选项。
- **卡片**：`vr_mi_problem_solution_001`

### 2. 观点—解释结构

- **结构**：观点/对策 → 解释说明/举例论证
- **适用场景**：前面已有明确观点，后文用例子、原因、背景反复支撑。
- **重点**：观点或对策。
- **不适用情况**：后文转折提出了新观点。
- **常见错误**：把解释、例子、背景当成中心。
- **卡片**：`vr_mi_view_explain_001`

### 3. 转折结构

- **结构**：前文铺垫/让步 → 转折后提出问题、观点或对策
- **触发词**：但是、然而、不过、可是、其实。
- **重点**：转折后；若转折后又给出对策，则最终重点是对策。
- **常见错误**：选转折前背景。
- **卡片**：`vr_mi_turning_point_001`

### 4. 因果结构

- **结构一**：原因 → 结果/观点。
- **结构二**：结果/问题 → 原因。
- **触发词**：因为、由于、因此、所以、根源在于、这是因为。
- **重点**：
  - 果因结构重点在原因；
  - 因果推出对策时重点在结论/对策。
- **常见错误**：文段只分析原因，选项却给出无依据解决方案。
- **卡片**：`vr_mi_causality_001`

### 5. 必要条件结构

- **结构**：只有/关键/基础/前提/必须/需要 → 对策条件
- **重点**：必要条件对应的措施或关键条件。
- **常见错误**：把条件和结果方向读反。
- **卡片**：`vr_mi_necessary_condition_001`

### 6. 总分 / 分总结构

- **结构**：总句 → 分述，或分述 → 总句。
- **重点**：总句。
- **常见错误**：选择分述中的某一个细节。
- **卡片**：`vr_mi_total_part_001`

### 7. 并列结构

- **结构**：多个并列层次共同说明一个主题。
- **适用场景**：时间、地区、人物、方面并列。
- **重点**：整体概括；时间轴常概括为“发展历程”。
- **常见错误**：只选并列中的一项。
- **卡片**：`vr_mi_parallel_001`

### 8. 背景—新发现结构

- **结构**：背景知识/旧认识 → 新发现/新观点
- **重点**：新发现。
- **常见错误**：选背景定义、旧知识。
- **卡片**：`vr_mi_background_new_discovery_001`

## 关键信息方法

### 主体词 / 主题词

- **适用场景**：选项主语差异明显，或文段多次出现同一核心对象。
- **操作**：圈首句主语、高频词、重点句主语。
- **排除**：主语不一致、主题词错位、范围扩大/缩小。
- **卡片**：`vr_mi_subject_word_001`

### 主题句

- **适用场景**：文段出现观点、对策、结论句。
- **操作**：找“因此、所以、为此、应该、需要、关键、基础”等标志后的句子。
- **排除**：来自背景、例子、分析的选项。
- **卡片**：`vr_mi_topic_sentence_001`

### 尾句重点

- **适用场景**：尾句是观点、对策或结论。
- **不适用情况**：尾句只是例子、补充材料或并列末项。
- **卡片**：`vr_mi_end_sentence_focus_001`

## 标题选择方法

### 标题拟定总流程

- **适用场景**：题干问“最适合做标题”。
- **步骤**：
  1. 先按主旨题确定文段中心。
  2. 标题必须包含核心主题词。
  3. 标题要准确、简洁、醒目。
  4. 排除过泛、过细、不准确、缺主题词的选项。
- **卡片**：`vr_mi_title_total_001`

### 标题关键词覆盖

- **适用场景**：标题选项之间主题词不同。
- **操作**：标题主体词必须与文段核心主体一致。
- **卡片**：`vr_mi_title_keyword_001`

## 选项排除体系

| 干扰项类型 | 判定规则 | 对应卡片 |
|---|---|---|
| 出处不对 | 来自背景、例子、分析、转折前、因此前，且不是重点层 | `vr_mi_eliminate_source_001` |
| 偷换概念 | 主语、对象、动作关系、方向发生变化 | `vr_mi_eliminate_concept_shift_001` |
| 范围扩大/缩小 | 选项范围大于或小于文段中心范围 | `vr_mi_eliminate_scope_001` |
| 片面 | 并列结构只覆盖其中一项 | `vr_mi_eliminate_scope_001` |
| 绝对化 | 出现“最、唯一、全部、完全”等且原文无依据 | `vr_mi_eliminate_absolute_001` |
| 无中生有 | 原文未提及且无法从结构推出 | `vr_mi_eliminate_none_001` |
| 下定义误选 | 文段是观点/对策，选项却是中性定义 | `vr_mi_eliminate_definition_001` |
| 杂糅 | 选项拼凑原文不同层次词语，逻辑关系不成立 | `vr_mi_eliminate_splice_001` |
| 比较无据 | 文段没有双主语平等比较，选项强行比较 | `vr_mi_eliminate_compare_001` |
| 太中性 | 文段有问题/态度，选项只是泛泛介绍 | `vr_mi_eliminate_neutral_001` |

## 考场优先级

1. `vr_mi_total_flow_001`：任何主旨意图题先调用。
2. `vr_mi_question_router_001`：先判题干问法。
3. `vr_mi_subject_word_001`：锁定主体词，防偷换。
4. 有对策时调用 `vr_mi_problem_solution_001` / `vr_mi_intention_solution_001`。
5. 有转折时调用 `vr_mi_turning_point_001`。
6. 有因果或原因解释时调用 `vr_mi_causality_001`。
7. 有并列时调用 `vr_mi_parallel_001`。
8. 标题题调用 `vr_mi_title_total_001`。
9. 最后统一调用选项排除卡片组，完成验证。

## 易错点

- 没有分析结构，直接凭语感选。
- 把背景、例子、分析当成中心。
- 原文有对策，却选问题项。
- 主体词偷换。
- 并列结构选片面项。
- 因果结构中把结果当原因，或把原因当对策。
- 选项出现“最、完全、唯一”等绝对化表达但原文没有支撑。
- 标题题只看是否醒目，不看是否准确覆盖中心。
