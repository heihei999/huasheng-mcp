# 图形推理视觉检查清单补充（audited）

## 0. 文件定位

本文件来源于对以下两个图推 PDF 的人工视觉核对：

```text
花生图形推理笔记.pdf
【判断推理】图形推理.pdf
```

本文件只补充视觉观察清单，不替代：

```text
knowledge_base/all_cards.jsonl
knowledge_base/module_originals/graphic_reasoning/graphic_reasoning_cards_audited.jsonl
```

适用场景：

```text
1. 后续构建 graphic_reasoning v0.1 method scaffold
2. 给多模态大模型提供图推观察顺序
3. 约束多模态大模型必须先看视觉证据，再给候选规律
4. 无法唯一时输出 analysis_only，不强行选答案
```

不适用场景：

```text
1. 不用于本地图像识别
2. 不用于 OCR
3. 不用于 OpenCV / PIL / cv2 视觉解析
4. 不用于自动数点线面
5. 不用于训练机器学习模型
6. 不用于根据题号或标准答案写规则
```

## 1. 图推总观察顺序

```text
第一层：命题形式
第二层：组成关系
第三层：属性规律
第四层：数量规律
第五层：位置规律
第六层：样式规律
第七层：特殊题型
第八层：空间类题型
第九层：选项验证
第十层：不确定性约束
```

命题形式：

```text
一组图
两组图
九宫格
分组分类
类比式图推
空间重构
```

命题形式说明：

```text
一组图：按从左到右寻找递推、周期、数量或位置变化。
两组图：先看第一组内部规律，再迁移到第二组。
九宫格：优先横向看，再纵向看，必要时看 S 型、米字型或中心对称关系。
分组分类：先找每组内部共同点，再找两组之间差异点。
空间重构：优先看相对面、公共边、公共点和选项可排除点。
```

---

## 2. 组成关系路由

```text
组成相同 → 优先看位置规律：平移、旋转、翻转、移动路径、方向变化、内外位置变化。
组成相似 → 优先看样式规律：遍历、加减同异、去同存异、去异存同、黑白运算、局部替换。
组成不同 → 优先看属性和数量：点、线、面、角、素、一笔画、封闭开放、对称、曲直。
特殊图形 → 优先触发专项检查：黑白块、汉字、数字、字母、功能元素、六面体展开图、截面图、三视图、立体拼合。
```

约束：

```text
组成相同时，不要一上来数点线面；优先检查位置变化。
组成相似时，不要一上来数数量；优先检查样式变化。
组成不同时，再系统检查属性和数量。
```

## 3. 十五字口诀

```text
屈臣氏整风
点线面角素
直接想位移
```

解释：

```text
屈：曲线 / 直线
臣：对称
氏：相同 / 相似图形
整：整体 / 部分
风：封闭 / 开放

点：交点、端点、切点、特殊点
线：直线、曲线、线段、特殊线组
面：封闭空间、最大面、最小面、部分面
角：直角、锐角、钝角、角数量
素：元素种类、元素个数、元素换算

直：垂直、平行
接：连接方式，包括点连接、线连接、面连接
想：方向
位：位置
移：平移、旋转、翻转
```

## 7. 样式规律视觉检查

### 7.1 遍历

```text

```

### 7.2 加减同异

```text
相加
相减
去同存异
去异存同
叠加后变色
叠加后抵消
同位置运算
```

约束：

```text
必须逐位置比较。
必须先从已知图推出统一规则。
不能凭单个位置猜运算结果。
若运算表不能统一解释已知图，应 analysis_only。
```

### 7.3 黑白运算

流程：

```text
1. 判断是否为同位置黑白叠加。
2. 建立黑 + 黑、黑 + 白、白 + 黑、白 + 白的运算表。
3. 用至少一组已知图验证运算表。
4. 运算表一致后再代入问号图。
5. 若题干无法推出唯一运算表，不强行选择。
```

强调：

```text
不同题的黑白运算表可能不同，不能预设固定运算规则。
```

## 10. 黑白块视觉检查清单

```text
整体对称
黑白面积
黑块数量
白块数量
黑块位置

黑块连通
白块连通
黑块相邻关系
黑白相邻关系
同位置运算
局部平移
局部旋转
```

推荐顺序：

```text
1. 先看整体对称性。
2. 再看黑白面积比例。
3. 再看黑块数量是否递增、递减或恒定。
4. 再看黑块是否连通，连通块数量是否变化。
5. 再看黑块是否相邻，边相邻还是角相邻。
6. 再看同位置黑白运算。
7. 最后看局部位置移动。
```

约束：

```text
黑白块题不能只看黑块数量；若数量无规律，应继续检查连通、相邻、对称、运算和位置。
```

## 14. 六面体展开图视觉检查清单

检查：

```text
相对面
公共边
公共点
时针方向
面内图案方向
特殊面
选项排除
```

展开图结构：

```text
1-4-1
2-3-1
2-2-2
0-3-3
```

提示：

```text
六面体展开图常见 4 类 11 种结构。
```

相对面排除流程：

```text
1. 找出展开图中必然相对的面。
2. 若选项中把相对面放成相邻面，直接排除。
3. 若两个相对面同时出现在可见三面中，直接排除。
```

公共边验证流程：

```text
1. 找三个面中两两相邻的公共边。
2. 检查折叠后公共边两侧图案方向是否一致。
3. 特殊展开图原图不重要时，重点盯选项公共边。
```

公共点验证流程：

```text
1. 找三个面共同交汇点。
2. 检查图案是否都朝向同一公共点。
3. 若公共点关系不一致，排除。
```

时针法流程：

```text
1. 选三个相邻面。
2. 按固定顺序标记三个面。
3. 比较展开图与选项折叠后的时针方向。
4. 方向相反则排除。
```

约束：

```text
若仅靠相对面无法排除，应继续使用公共边、公共点、时针法，不能直接猜。
```

## 15. 截面图视觉检查清单

检查：

```text
立体类型
切入位置
切割方向
截面边数
截面形状
能否截出选项形状
是否存在不可能截面
```

常见规则：

```text
1. 六面体一般最多可截出六边形。
2. 六面体不能截出正五边形。
3. 圆柱常见截面包括圆、椭圆、矩形等。
4. 圆锥常见截面包括圆、椭圆、三角形等。
5. 圆锥通常不能截出四边形。
6. 考试中不必细分抛物线、双曲线；一般上面为弧线、下面为直线即可。
```

长方体提示：

```text
1. 从点入刀斜切，可出现三角形类截面。
2. 从面入刀斜切，可出现梯形类截面。
3. 垂直切割，可出现矩形类截面。
```

约束：

```text
如果选项形状需要曲面参与，必须确认原立体是否有曲面。
```

## 16. 三视图视觉检查清单

检查：

```text
主视图
俯视图
左视图
长对正
高平齐
宽相等
可见线
不可见线
遮挡关系
曲面交接
```

核心规则：

```text
1. 长对正。
2. 高平齐。
3. 宽相等。
4. 有线就有线，没线就没线。
5. 被遮挡部分不可见。
6. 曲面平滑交接处一般无线。
```

流程：

```text
1. 先确定观察方向。
2. 再确定外轮廓。
3. 再检查内部可见线。
4. 最后检查遮挡线和曲面交接线。
```

## 17. 立体拼合视觉检查清单

检查：

```text
块数
体积
占地面积
最大块
特殊块
凹凸关系
长短关系
高低关系
分层结构
严丝合缝
```

推荐流程：

```text
1. 先数小方块总数。
2. 再找占地面积大的特殊块。
3. 按层画图或按层想象。
4. 检查凸凹是否互补。
5. 检查长短、高低是否一致。
6. 检查拼合后是否多块、少块或重叠。
```

约束：

```text
若某个选项不能严丝合缝，或需要重叠/穿插才能成立，应排除。
```

## 18. 选项验证流程

```text
1. 明确候选规律。
2. 用题干已知图验证该规律。
3. 若规律能解释全部已知图，再代入问号。
4. 对四个选项逐一验证。
5. 只有一个选项满足规律时才能输出答案。
6. 若多个选项满足或没有选项满足，应 analysis_only。
```

## 19. 不确定性约束

```text
1. 不得默认选择第一个选项。
2. 不得在无法唯一排除时强行给答案。
3. 不得用题号、case_id 或标准答案写规则。
4. 不得跳过视觉证据说明。
5. 不得只说"看起来像"，必须指出视觉依据。
6. 不得把图推 scaffold 当成本地图像 solver。
7. 不得引入 OCR、OpenCV、PIL 或机器学习依赖。
```

出现以下情况时，应输出 `analysis_only`：

```text
1. 规律无法统一解释题干图形。
2. 两个及以上选项都符合候选规律。
3. 必须依赖无法确认的小图细节。
4. 图形识别结果不可靠。
5. 空间折叠无法唯一排除。
6. 黑白运算表无法唯一确定。
```

## 20. 给多模态大模型的作答模板

```text
【命题形式】
说明是一组图、两组图、九宫格、分组分类、空间重构还是其他形式。

【组成判断】
说明图形组成是相同、相似、不同，还是特殊图形。

【优先规律】
根据组成判断，选择位置、样式、属性、数量、特殊或空间类路径。

【视觉证据】
逐条列出从图中看到的证据，例如数量、位置、方向、对称、曲直、开闭、黑白块、公共边等。

【候选规律】
提出最多 1-2 个候选规律，并说明为什么成立。

【选项验证】
逐一验证 A/B/C/D 是否符合候选规律。

【唯一性判断】
如果唯一，输出答案；如果不唯一，输出 analysis_only。

【不确定性说明】
说明哪些图形细节看不清，哪些规律不能唯一确定。
```

## 22. 验收口径

补充本文件后，可认为图推知识点达到以下状态：

```text
图推主方法卡：齐全
图推视觉细节清单：补齐
PDF/思维导图主框架：覆盖
作为 MCP 多模态 scaffold：足够
作为本地图像识别 solver：不适用
```

不应宣称：

```text
已经具备本地图形识别能力
已经能自动解析图片
已经能不借助多模态模型直接解图推
已经完整复刻 PDF 中每一个小图细节
```

正确表述：

```text
当前图推知识库已经具备构建多模态 method scaffold 的知识条件。
后续只需实现 scaffold 输出模板和测试，不需要改动主知识库与主 solver。
```
