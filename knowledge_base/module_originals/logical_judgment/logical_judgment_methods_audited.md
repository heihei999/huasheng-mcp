# 判断推理-逻辑判断方法库（audited）

## 题型总览

本模块面向“判断推理-逻辑判断”的文字类题目，核心处理对象不是零散知识点，而是“题干论证结构”。后续解题器应先调用 `lj_total_flow_001` 与 `lj_argument_structure_001`，再根据问法进入具体方法。

| 题型 | 典型问法 | 优先方法卡 |
|---|---|---|
| 加强 | 最能支持、最能加强、最能证明 | `lj_support_general_001`, `lj_support_bridge_001` |
| 削弱 | 最能削弱、最能质疑、最能反驳 | `lj_attribution_general_001`, `lj_weaken_evidence_conclusion_001` |
| 前提假设 | 需要补充的前提、结论成立还需基于 | `lj_premise_general_001` |
| 解释说明 | 最能解释上述现象/矛盾 | `lj_explanation_contradiction_001` |
| 评价论证 | 最能评价上述论证、取决于 | `lj_argument_evaluation_001` |
| 结论推出 | 由此可以推出、一定为真 | `lj_conclusion_translation_001` |
| 分析推理 | 真假话、范畴关系、日常条件匹配 | `lj_truth_value_analysis_001`, `lj_category_analysis_001`, `lj_daily_analysis_001` |

## 逻辑基本知识

### 1. 充分条件与必要条件

- **适用场景**：题干出现如果、只要、只有、除非、必须、需要、前提、不可或缺等条件词。
- **题干触发词**：如果、只要、若、所有都、只有、才、除非、否则、必须、必要前提。
- **不适用情况**：题干是因果归因、比例比较、解释矛盾时，条件翻译只作为局部辅助。
- **解题步骤**：
  1. 抽取 A、B。
  2. 判断条件词方向。
  3. 翻译为箭头式。
  4. 使用“肯前、否后”验证选项。
  5. 排除肯后、否前。
- **考场速算说明**：看到“必要、前提、必须、离不开”时，谁必要谁在箭头后。
- **常见错误**：把“只有 A 才 B”翻译成 A→B。

对应卡片：`lj_sufficient_necessary_001`, `lj_translation_if_001`, `lj_translation_only_001`, `lj_translation_unless_001`, `lj_inverse_contrapositive_001`

### 2. 或且关系与命题否定

- **适用场景**：题干出现 A 或 B、A 且 B、不同意、并非。
- **题干触发词**：或者、要么、且、并且、不赞成、不同意。
- **不适用情况**：问法是论证加强/削弱时，除非选项或论点含复合命题。
- **解题步骤**：
  1. 先写原命题。
  2. 整体否定。
  3. A或B 的否定为 -A且-B；A且B 的否定为 -A或-B。
  4. 检查选项是否完整表达否定。
- **常见错误**：只否定一个分支。

对应卡片：`lj_or_and_negation_001`

## 论证结构识别

### 1. 论点、论据、论证方式拆解

- **适用场景**：所有加强、削弱、前提、解释、评价论证题。
- **题干触发词**：因此、所以、由此可见、认为、推测、说明、导致、得益于。
- **不适用情况**：纯定义判断、图形推理、类比推理。
- **解题步骤**：
  1. 找论点：优先看结论标志词之后。
  2. 找论据：事实、数据、实验、调查、背景。
  3. 找论证关系：因果、比例、实验、条件、样本、解释矛盾。
  4. 找断点：论据到论点的跳跃处。
  5. 选项必须作用于论点、论据或断点。
- **考场速算说明**：加强找“补断点”，削弱找“拆断点”，前提找“没它不行”。
- **常见错误**：直接看选项，不拆论点和论据。

对应卡片：`lj_total_flow_001`, `lj_argument_structure_001`

## 加强类方法

### 1. 支持题主体话题一致原则

- **适用场景**：问“最能支持/加强/证明”。
- **题干触发词**：最能支持、最能加强、最能证明。
- **不适用情况**：问“前提”时要额外做否定代入；问“解释”时要解释矛盾双方。
- **解题步骤**：
  1. 找论点。
  2. 找论据。
  3. 判断选项主体是否一致。
  4. 判断选项话题是否一致。
  5. 比较是否直接补断点。
- **考场速算说明**：主体一致、话题一致、贴合题干是所有论证题的大前提。
- **常见错误**：选常识正确但与题干无关的选项。

对应卡片：`lj_support_general_001`

### 2. 断点搭桥

- **适用场景**：论据关键词 A 与论点关键词 B 不同。
- **题干触发词**：因此、说明、由此认为。
- **不适用情况**：题干已经直接证明 A 与 B，无断点。
- **解题步骤**：
  1. 找 A：论据核心。
  2. 找 B：论点核心。
  3. 找 A 到 B 的缺口。
  4. 选择把 A 和 B 连接起来的选项。
- **考场速算说明**：桥接项通常强于泛泛事实支持。
- **常见错误**：只加强 A，没有连接 B。

对应卡片：`lj_support_bridge_001`

### 3. 加入论据、解释机制、必要条件、自断一臂

- **加入论据和事实**：选项补充同主体、同话题、同方向事实。
- **解释机制**：选项解释为什么某因素会导致某结果。
- **必要条件**：选项提供结论成立不可缺少的条件。
- **自断一臂**：选项排除一个会使题干路径失效的替代路径。

对应卡片：`lj_support_add_evidence_001`, `lj_support_explain_mechanism_001`, `lj_support_necessary_condition_001`, `lj_support_self_block_001`

## 前提假设类方法

### 1. 没它不行检验法

- **适用场景**：问“需要补充的前提”“结论成立还需基于”。
- **题干触发词**：前提、假设、依据、还需基于、需要补充。
- **不适用情况**：普通加强题中，不能把所有支持项都当成前提。
- **解题步骤**：
  1. 找论点、论据、断点。
  2. 对选项做否定代入。
  3. 否定后论证崩塌，则是前提。
  4. 否定后只是变弱但仍可成立，则不是最优前提。
- **考场速算说明**：“选没它不行，不选有它更好”。
- **常见错误**：选了“有它更好”的普通支持项。

对应卡片：`lj_premise_general_001`

### 2. 前提三类高频方法

- **断点搭桥**：补上论据与论点之间缺失关系。
- **补充漏洞**：题干承认直接证据缺失，选项补足缺失条件。
- **能与不能**：题干反应或结论依赖某行为可实现，选项说明可行性。

对应卡片：`lj_premise_bridge_001`, `lj_premise_fill_gap_001`, `lj_premise_enable_condition_001`

## 削弱质疑类方法

### 1. 一般质疑

- **适用场景**：无论据有结论、有论据有结论、严格逻辑关系。
- **题干触发词**：削弱、质疑、反驳、不能支持、论证不成立。
- **不适用情况**：解释说明题。
- **解题步骤**：
  1. 无论据有结论：找有理由的反结论。
  2. 有论据有结论：肯定论据，否定结论。
  3. 严格条件关系：翻译成 A→B，用 A且非B 反驳。
- **考场速算说明**：削弱不是“说反话”，而是让论据推不出论点。
- **常见错误**：只攻击无关背景事实。

对应卡片：`lj_weaken_general_no_premise_001`, `lj_weaken_evidence_conclusion_001`, `lj_weaken_strict_conditional_001`

### 2. 归因论证

- **适用场景**：题干由相关、同时出现、实验差异推出 A 导致 B。
- **题干触发词**：导致、影响、得益于、原因是、归因于、因此认为。
- **不适用情况**：纯条件推出题。
- **解题步骤**：
  1. 抽取原因 A 与结果 B。
  2. 判断题型：对比实验、一般归因、直接根本原因、对策隐含归因。
  3. 优先检查：另有他因、因果倒置、否定此因、根表原因。
  4. 若是实验，再检查初始状态、对照组、样本。
  5. 选择最直接破坏 A→B 的选项。
- **考场速算说明**：题干只证明相关，选项能证明不是 A 导致 B 就强。
- **常见错误**：把相关性当因果。

对应卡片：`lj_attribution_general_001`, `lj_weaken_alternative_cause_001`, `lj_weaken_reverse_causation_001`, `lj_weaken_not_this_cause_001`, `lj_weaken_root_surface_cause_001`

### 3. 对比实验问题

- **适用场景**：实验组/对照组、处理变量、结果变量。
- **题干触发词**：实验、研究人员、两组、对照、测试者、小鼠、样本。
- **不适用情况**：非实验归因。
- **解题步骤**：
  1. 找实验组和对照组。
  2. 找处理变量和结果变量。
  3. 检查初始状态是否一致。
  4. 检查是否缺少对照组或对照不当。
  5. 检查样本是否代表总体。
- **常见错误**：把实验后差异当作实验前一致。

对应卡片：`lj_experiment_initial_state_001`, `lj_experiment_missing_control_001`, `lj_compare_experiment_construct_001`, `lj_weaken_invalid_sample_001`

## 比例类与解释说明

### 1. 比例类论证

- **适用场景**：题干用绝对数比较后推出风险、威胁、概率、比例大小。
- **题干触发词**：人数更多、死亡人数更少、销量更高、比例、率、占比、风险。
- **不适用情况**：题干已经给出同分母下的比例。
- **解题步骤**：
  1. 判断论点是否比较率。
  2. 拆分子。
  3. 找分母。
  4. 若题干只比分子，则找分母差异选项。
  5. 不能用绝对数直接推出比例大小。
- **常见错误**：只比较死亡人数、患病人数、销量等分子。

对应卡片：`lj_proportion_argument_001`

### 2. 解释说明

- **适用场景**：题干给出看似矛盾的两个现象。
- **题干触发词**：最能解释、解释上述现象、解释矛盾。
- **不适用情况**：题干要求支持或削弱结论。
- **解题步骤**：
  1. 找矛盾双方 A 与 B。
  2. 找能同时解释 A 和 B 的合理事实。
  3. 排除只解释一方的选项。
- **常见错误**：把解释题做成加强题。

对应卡片：`lj_explanation_contradiction_001`

## 结论推出与分析推理

### 1. 结论推出

- **适用场景**：由此可以推出、一定为真。
- **核心方法**：条件翻译、逆否等价、或且否定、范畴关系。
- **考场要求**：只选必然推出，不选可能为真。

对应卡片：`lj_conclusion_translation_001`, `lj_conclusion_equivalent_inference_001`

### 2. 分析类

- **真假分析**：找矛盾、真假数量、假设代入。
- **范畴分析**：画集合圈，保留不确定区域。
- **日常分析**：对象多用列表，顺序位置用画图，选项少用代入。

对应卡片：`lj_truth_value_analysis_001`, `lj_category_analysis_001`, `lj_daily_analysis_001`

## 常见无关选项

1. 主体不一致。
2. 话题不一致。
3. 只说背景常识，不改变论证关系。
4. 只重复题干。
5. 偷换概念或扩大范围。
6. 只说明相关，不说明因果。
7. 只比较分子，不比较比例。
8. 只解释一半矛盾。

对应卡片：`lj_irrelevant_options_001`

## 加强/削弱力度比较

### 加强力度大致顺序

1. 断点搭桥 / 必要条件 / 排除关键漏洞。
2. 直接增加同主体同话题事实。
3. 解释机制。
4. 举例或类比。
5. 泛泛背景。
6. 无关项。

### 削弱力度大致顺序

1. 严格条件反例：A且非B。
2. 因果倒置、另有他因、否定此因。
3. 实验初始状态、缺少对照组、样本无效。
4. 肯定论据但否定结论。
5. 少数质疑论据。
6. 无关项。

对应卡片：`lj_strength_order_compare_001`
