# 判断推理-定义判断与类比推理方法库（复审版）
## 题型总览
- **定义判断**：题干给出概念定义，要求判断选项属于/不属于/符合/不符合定义。核心是先理解被定义词，再核对主体、对象、条件、方式、目的、结果和特殊要求。
- **类比推理**：题干给出词项关系，要求在选项中找最相似关系。核心是先用大众思维识别一级关系，一级不够再做二级辨析。
## 定义判断方法
### 定义判断总体理解流程

- 不适用情况：题目要求比较两个词语关系时优先路由到类比推理；题干没有完整定义、只让判断结论能否推出时不调用本卡

- 解题步骤：
  - 先看问法，明确是“属于”还是“不属于”，不属于题最大陷阱是把符合定义的例子当答案。
  - 抓住被定义词，先根据概念名称判断本质解释和核心方向。
  - 整体理解定义所指的人、事或现象，必要时联系生活常识，但必须回到题干验证。
  - 识别有效信息：主体、客体/对象、条件、方式、目的、结果、特殊要求。
  - 看选项时不要因某一特点就排除，重点关注被定义词中“不包含”的信息。
  - 技巧题可用同构选项同时排除、对比枚举型逐一定义对应、涉及时间变化时核对前后状态。
  - 最终输出每个选项的符合/不符合要素，不允许只给答案。
- 常见错误：只见树木不见森林：把定义拆成过多碎片，反而忽略被定义词本身。；属于/不属于方向看反。；把生活常识中的相似现象误当成定义项。；题干另有相近概念或相似焦点时，抓手常藏在本质解释或延伸解释里。；有限枚举时可做抓手，无限枚举不可当抓手。

### 属于/不属于方向控制法
- method_id: `dj_belong_not_belong_001`
- 适用场景：题干问属于/不属于/符合/不符合/最符合/最不符合；选项中多个看似符合定义
- 不适用情况：题目不是定义判断，或问法不涉及符合/不符合定义
- definition_elements：{"问法": "属于/不属于必须先定", "主体": null, "客体/对象": null, "条件": null, "方式": null, "目的": null, "结果": null, "特殊要求": null}
- 解题步骤：
  - 先在草稿中写明本题要找“符合”还是“不符合”。
  - 属于题：排除不符合定义项，保留最符合项。
  - 不属于题：先确认多个符合项可作为理解定义的例子，再寻找唯一不符合项。
  - 最后核对被选项是否与问法方向一致。
- 常见错误：不属于题误选符合定义的典型项。；看完选项后忘记回到题干问法。

### 抓住被定义词法
- method_id: `dj_defined_term_001`
- 适用场景：定义名称本身含有明显语义，如“垂直搜索引擎”“亲情经济”“工具性求助”；选项与定义细节接近但概念本质不同
- 不适用情况：被定义词为高度抽象、名称不能直接提示核心特征时，不可只靠名称判断
- definition_elements：{"被定义词": "核心入口", "主体": null, "客体/对象": null, "条件": null, "方式": null, "目的": null, "结果": null, "特殊要求": null}
- 解题步骤：
  - 先读被定义词名称，标出名称中最关键的一两个词。
  - 判断这些词为何被用作定义之名，推测其最核心特征。
  - 回到定义文本验证核心特征。
  - 选项代入时优先排除与名称核心特征不符的项。
  - 最后再核对定义中的主体、对象、特殊要求。
- 常见错误：只看定义句中的局部词，不看定义名称。；把平台、网站、查询行为误等同于搜索引擎。

- method_id: `dj_option_setting_001`
- 适用场景：定义判断题进入选项比对阶段；选项相近、题干信息较多、不确定哪个要素是核心
- 不适用情况：题目为类比词项关系题时不调用；题干没有明确定义时不调用
- definition_elements：{"问法": "必须先读", "被定义词": "必须检查", "主体": "按题干检查", "客体/对象": "按题干检查", "条件": "按题干检查", "方式": "按题干检查", "目的": "按题干检查", "结果": "按题干检查"}
- 解题步骤：
  - 看提问：先圈出“属于/不属于/符合/不符合”，这是最大陷阱。
  - 看题干：识别有效信息，优先找准关键词、关键句。

### 定义关键词查找抓手法
- method_id: `dj_keyword_lookup_001`
- 适用场景：定义文本较长，难以确定核心条件；题干出现名词本质解释、延伸解释、逻辑关联语气词、必要条件、否定条件或枚举结构
- 不适用情况：定义很短且核心要素清晰时，不必过度拆解；无限枚举或开放式举例不能单独作为抓手
- definition_elements：{"关键词/关键句": "核心", "条件": "强约束", "否定条件": "强排除", "有限枚举": "可作抓手"}
- 解题步骤：
  - 先找题干既有定义名词的本质解释和延伸解释。
  - 若定义中出现逻辑关联语气转折，尤其形成对比色彩，优先作为抓手。
  - 若出现必要性条件型语句，将其视为强约束。
  - 若出现否定性条件型语句，将其视为强排除条件。
  - 若定义用有限枚举界定概念，可作为抓手；若是无限枚举，只能作为辅助理解。
- 常见错误：把普通修饰词当成强条件。；把无限枚举当作穷尽条件。；看不到否定条件导致反选。
- 来源：花生判断笔记总结.pdf p.20

### 主体限定排除法
- method_id: `dj_subject_check_001`
- 适用场景：定义中出现明确主体，如年轻人、商家、求助者、工作学习者等；选项行为主体明显不同
- 不适用情况：定义没有限定主体，或主体只是泛指对象时，不要强行排除
- definition_elements：{"主体": "必须匹配", "客体/对象": "后续检查", "条件": null, "方式": null, "目的": null, "结果": null, "特殊要求": null}
- 解题步骤：
  - 标记定义中的行为主体或适用主体。
  - 逐项识别选项中的行为主体。
  - 主体不符合时直接排除；主体符合后再看对象和行为。
  - 若题问不属于，主体不符的选项优先成为候选答案。
- 常见错误：忽略年龄、身份、组织类型等主体限定。

- 不适用情况：定义只描述行为过程，没有明确对象限制时，不要硬套对象检查
- definition_elements：{"主体": "先检查", "客体/对象": "必须匹配", "条件": null, "方式": null, "目的": null, "结果": null, "特殊要求": null}
- 解题步骤：
  - 标出定义中行为作用的对象。
  - 识别选项中实际被看待、被求助、被作用的对象。

  - 对象符合后再检查方式和目的。

- 适用场景：定义中出现时间、场景、对象范围、关系类型等限定；选项主体行为相似但缺少限定条件
- 不适用情况：定义没有限定条件，只有一般描述时，不要凭常识添加特殊要求

- 解题步骤：
  - 圈出定义中的特殊要求，例如传统节日期间、工作学习过程中、借助他人等。

- method_id: `dj_multi_definition_001`
- 适用场景：题干同时给出两个或多个定义；问属于其中某一种定义；定义之间有明显分类差异
- 不适用情况：只有单一定义时不必使用多定义对照表
- definition_elements：{"主体": "逐定义提取", "客体/对象": "逐定义提取", "条件": "逐定义提取", "方式": "逐定义提取", "目的": "逐定义提取", "结果": "逐定义提取", "特殊要求": "逐定义提取"}
- 解题步骤：
  - 确认题干是否给出两个或多个并列定义。
  - 分别抽取每个定义的主体、对象、行为方式、目的或结果。

- 适用场景：定义中出现“用/通过/借助/以达到/以利于/从而/结果”等链条词；选项行为相似但目的或结果不同
- 不适用情况：题干没有方式、目的或结果链条时，不要强行拆链
- definition_elements：{"主体": "辅助", "客体/对象": "辅助", "条件": "辅助", "方式": "重点检查", "目的": "重点检查", "结果": "重点检查", "特殊要求": "辅助"}
- 解题步骤：
  - 从定义中提取方式、目的、结果三个位置。

### 时间前后与变化对比法
- method_id: `dj_time_change_001`
- 适用场景：定义或选项涉及时间前后、状态变化、过程变化；选项设置存在“以前情况”和“现在情况”对比
- 不适用情况：定义只描述静态属性且无时间变化信息时不优先调用
- definition_elements：{"条件": "时间前后", "方式": "变化过程", "结果": "变化后状态"}
- 解题步骤：
  - 定位定义中的时间词或变化词。
  - 区分初始状态、过程行为、变化后结果。
  - 检查选项是否同时满足前后状态或变化方向。
  - 若只描述某一时间点而缺失变化过程，谨慎排除。
- 常见错误：只看现在状态，忽略是否发生了定义要求的变化。；把时间点错位当成符合定义。

## 类比推理方法
### 类比推理总体解题流程
- method_id: `ar_total_flow_001`
- relation_type: `总流程`
- 适用场景：题干呈现词语对、三词组、括号式类比；问与题干关系最为相似
- 不适用情况：题干给出概念定义并问属于/不属于时路由到定义判断
- secondary_check：["大众思维/简单思维", "一级关系优先", "一级不够再二级辨析", "关系方向一致", "主体与数量一致"]
- 解题步骤：
  - 先用大众思维、简单思维判断题干词项之间的直接内在联系。
  - 优先想“造句子”：选择相似点和题干最多的关系，而不是强行找冷僻常识。
  - 先粗分为语义、语法、逻辑、对应、拆词等一级关系。
  - 一级辨析选不出时，再进入二级辨析；不要一开始就陷入细枝末节。
  - 二级辨析依次检查：感情色彩、程度轻重、词性、主动/被动、种属/组成、矛盾/反对、特指/泛指、关系方向。
  - 全同关系必须一点不能差；对应关系尤其要看主体、对象、数量、先后顺序和方式目的。
- 常见错误：想太复杂，使用冷门知识。；只看表面同类词，不看内在关系。
- 来源：【判断推理】类比推理.pdf p.1; 花生判断笔记总结.pdf p.21,22

### 二级辨析总控法
- method_id: `ar_secondary_check_001`
- relation_type: `二级辨析`
- 适用场景：多个选项都满足一级关系；题干关系看似简单但选项相近
- 不适用情况：一级关系已唯一确定答案时，不必过度二级辨析
- secondary_check：["种属 vs 组成", "矛盾 vs 反对", "特指 vs 泛指", "方向一致", "词性一致"]
- 解题步骤：
  - 先确认所有候选项是否满足同一一级关系。
  - 若为包含关系，区分种属与组成。
  - 若为互斥关系，区分矛盾与反对。
  - 若为包含/种属关系，区分特指与泛指。
  - 补充检查词性、方向、关系强弱和对象类别。
  - 选择二级关系与题干最一致的选项。
- 常见错误：一级关系一致就匆忙选项。；忽略方向与层级。
- 来源：【判断推理】类比推理.pdf p.1; 花生判断笔记总结.pdf p.21,22

### 语义关系感情色彩与程度二级辨析法
- method_id: `ar_semantic_secondary_001`
- relation_type: `语义关系`
- 适用场景：题干为近义、反义或比喻/象征关系，多个选项一级关系均成立；选项近义反义关系相似但褒贬或程度不同
- 不适用情况：题干主关系不是语义关系时，不优先调用
- secondary_check：["感情色彩:褒义/贬义/中性", "程度:好感/喜欢/热爱", "词性一致"]
- 解题步骤：
  - 先确认题干是近义、反义、比喻义、象征义或引申义。
  - 若是近义/反义，继续比较感情色彩：褒义、贬义、中性。
  - 再比较程度轻重：好感、喜欢、热爱等强度是否一致。
  - 若是比喻/象征义，检查词性是否一致。
  - 只有一级关系和二级属性都一致时才保留选项。
- 常见错误：只看近义反义，不看褒贬色彩。；忽略程度强弱差异。

### 对应关系主体数量核对法
- method_id: `ar_correspond_subject_quantity_001`
- relation_type: `对应关系`
- 适用场景：题干为功能、时间顺序、方式目的、职业场所、工具对象等对应关系；多个选项均能造句但关系细节不同
- 不适用情况：题干只是语义或语法关系时不优先调用
- secondary_check：["功能主次", "时间先后", "主体一致", "主体数量一致", "方式是否达成目的"]
- 解题步骤：
  - 先确认对应关系类型：功能、时间顺序、方式目的、职业场所、工具对象等。
  - 若为功能，区分主要功能与次要功能。
  - 若为时间顺序，先看时间先后，再看主体是否一致、主体数量是否一致。
  - 若为方式目的，看通过某种方式是否能达到某种目的，并核对主体是否一致。
  - 选项能造句不代表一定正确，还要检查主体、对象、数量和方向。

### 近义关系识别法
- method_id: `ar_semantic_synonym_001`
- relation_type: `语义关系-近义`
- 适用场景：题干两个词语意思相近或可互相解释；选项中存在近义词组
- 不适用情况：词语只是同领域、同类别但语义不近时不要判近义
- secondary_check：["词性一致", "感情色彩一致", "语义强弱一致"]
- 解题步骤：
  - 判断题干两词是否可以近义替换。
  - 排除只是在情境上相关但语义不近的选项。
  - 若多个选项近义，进一步看感情色彩、词性和语体是否一致。
  - 选出近义程度和词性最一致的项。
- 常见错误：把相关关系误判为近义。；忽略褒贬色彩。

### 反义关系识别法
- method_id: `ar_semantic_antonym_001`
- relation_type: `语义关系-反义`
- 适用场景：题干两个词语意义相反或方向相反；选项中存在明显反义词组
- 不适用情况：两个概念可以并存或有第三种可能时，不一定是矛盾关系，应考虑反对关系
- secondary_check：["词性一致", "反义方向一致", "是否属于矛盾/反对"]
- 解题步骤：
  - 判断题干是否是语义层面的反义。
  - 识别选项中是否也为反义，而非因果、并列或对应。
  - 若涉及非A即B，转入逻辑关系中的矛盾/反对二级辨析。
  - 比较反义强度、词性和结构。
- 常见错误：把反义与逻辑矛盾混同。；忽略有无第三种可能。

### 种属关系识别法
- method_id: `ar_logic_species_genus_001`
- relation_type: `逻辑关系-种属`
- 适用场景：A是B的一种；可表达为所有A都是B
- 不适用情况：A只是B的一部分时是组成关系，不是种属
- secondary_check：["A是B的一种", "特指/泛指一致", "不是组成关系"]
- 解题步骤：
  - 用“A是B的一种/所有A都是B”测试题干。
  - 区分特指与泛指：苏格拉底-人类属于特指个体到类，公交车-汽车属于泛指类别到类。
  - 排除组成关系。
  - 检查选项是否同为种属，且特指/泛指层级一致。
- 常见错误：把轮胎-汽车误判为种属。；忽略特指/泛指差异。
- 来源：【判断推理】类比推理.pdf p.1; 花生判断笔记总结.pdf p.21

### 组成关系识别法
- method_id: `ar_logic_composition_001`
- relation_type: `逻辑关系-组成`
- 适用场景：A是B的一部分；题干存在零件-整体、成员-整体关系
- 不适用情况：若A是B的一种，应转入种属关系
- secondary_check：["A是B的一部分", "方向一致", "区别于种属"]
- 解题步骤：
  - 用“A是B的一部分”测试题干。
  - 确认A与B不是同类层级，而是部分与整体。
  - 检查选项是否也为部分-整体，且方向一致。
  - 若选项为一种-类别，排除。
- 常见错误：种属与组成混淆是类比三难点之一。
- 来源：【判断推理】类比推理.pdf p.1; 花生判断笔记总结.pdf p.21

### 反对/矛盾二级辨析法
- method_id: `ar_logic_opposition_contradiction_001`
- relation_type: `逻辑关系-反对/矛盾`
- 适用场景：题干两词互斥或相反；选项同时出现反对与矛盾关系
- 不适用情况：只是语义近反义但非概念集合关系时，不必套用矛盾/反对
- secondary_check：["有无第三者", "非A即B测试", "概念层级一致"]
- 解题步骤：
  - 先判断A与B是否互斥。
  - 若非A即B、没有第三者，则为矛盾关系。
  - 若有第三种可能，则为反对关系。
  - 选项必须与题干同为矛盾或同为反对。
  - 再检查词性和概念层级是否一致。
- 常见错误：把反对当矛盾，忽略第三者。
- 来源：【判断推理】类比推理.pdf p.1; 花生判断笔记总结.pdf p.21

### 交叉关系识别法
- method_id: `ar_logic_cross_001`
- relation_type: `逻辑关系-交叉`
- 适用场景：两个概念部分重合：有的A是B，有的B是A；选项中出现类别交叉关系
- 不适用情况：若所有A都是B则为种属，不是交叉；若A是B一部分则为组成
- secondary_check：["有的A是B", "有的B是A", "非种属", "非组成"]
- 解题步骤：
  - 判断是否存在A∩B且A不完全属于B、B不完全属于A。
  - 用“有的A是B，有的B是A”测试。
  - 排除种属、组成、全同关系。
  - 比较选项是否也为交叉关系。
- 常见错误：把交叉误判为并列或种属。
- 来源：【判断推理】类比推理.pdf p.1; 花生判断笔记总结.pdf p.21

### 因果关系识别法
- method_id: `ar_correspond_cause_effect_001`
- relation_type: `对应关系-因果`
- 适用场景：题干一个词是原因，另一个词是结果；选项存在原因-结果结构
- 不适用情况：两个词只是时间先后或功能目的，不要误判为因果
- secondary_check：["因果方向一致", "区别于时间顺序", "区别于功能目的"]
- 解题步骤：
  - 判断A是否导致B，或B是否由A引起。
  - 明确因果方向。
  - 检查选项是否同方向因果。
  - 排除只是同时出现、先后发生、功能用途的选项。
- 常见错误：把时间顺序误判为因果。；因果方向倒置。

### 类比推理常见误判防错卡
- method_id: `ar_common_traps_001`
- relation_type: `防错复核`
- 适用场景：类比推理任意题最终复核；选项均有一定联系但难以取舍
- 不适用情况：非类比推理题不调用
- secondary_check：["大众思维", "简单思维", "种属/组成", "矛盾/反对", "特指/泛指", "方向一致"]
- 解题步骤：
  - 检查是否用冷门常识代替大众常识。
  - 检查是否把种属与组成混淆。
  - 检查是否把矛盾与反对混淆。
  - 检查是否把特指与泛指混淆。
  - 检查是否把因果、功能、目的、时间顺序混淆。
  - 检查是否只看词性或字数，未看内在关系。
- 常见错误：关系牵强也强行对应。
- 来源：【判断推理】类比推理.pdf p.1; 花生判断笔记总结.pdf p.21,22

## 考场调用优先级
1. 定义判断先调用 `dj_total_flow_001`，再按问法、被定义词、主体/对象/特殊要求、关键词抓手分流。
2. 类比推理先调用 `ar_total_flow_001`，优先判断一级关系；候选项相近时调用 `ar_secondary_check_001`。
3. 最终必须输出排除理由或关系比对理由，不允许只凭语感给答案。
